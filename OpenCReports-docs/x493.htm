Introduction

Expressions in OpenCReports are based on Flex lexer and Bison grammar that provide a solid and extensible basis for report expressions.

There are four data types in OpenCReports: numeric, string, datetime and error.

A numeric expression stores a high precision number. Most functions and operators deal with numbers.

A string expression stores a string literal. String can be concatenated or processed by string functions.

A datetime expression may store a date, a time (with or without timezone) or both. Also, it may store a time interval, e.g. 2 months that can be added to or subtracted from another datetime value.

An error expression is a special case of string: it stores a string literal (which is an error message) but it cannot be further processed by functions or operators. Instead, an error is propagated up from subexpressions to the final result of the expression.
