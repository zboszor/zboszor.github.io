The predecessor: RLIB

The idea to write OpenCReports started with my getting acqauinted with RLIB in 2005 and working with it (and on it) for a very long time, with the original implementors finally losing interest in developing RLIB further.

RLIB is a report generator library, so is OpenCReports.

This means that using a tabular data source (for example an array with labels for the column names, or an SQL query) and a description file that specifies how to display the data, the input data is transformed into various output formats, some for human viewing, some for further machine processing. Such output formats may be PDF, HTML, XML, plain text or CSV.

To overcome some of the shortcomings seen in the predecessor, the ideas of RLIB were used for a completely new implementation with high level of compatibility to the original.

The name OpenCReports came from the fact that it's written in the C programming language and is implemented in an open way and using an free software license.
